Booker Types
  • Agreement Booker — pre-created, linked to Membership or MOU. Has allowance, allowedLocations, bookingCategories. Access via magic link → /booker/portal/:token
  • General Public Booker — anyone. No agreement. Access via /book/:userId or /casual-hire. Sees full pricing.
Location Rules
  • allowedLocations[] on agreement — enforced server-side on booking create
  • allowedVenueIds[] on MOU — more granular than location names
  • bookingCategories[] on agreement — controls which space types booker can access (office / studio / etc)
  • isFirstBooking flag on booking record — exists in schema
  • Locations cannot be mixed in a single booking session — enforced server-side, not surfaced clearly in portal UI
What Works
  • Change requests (booker-initiated)
  • Pricing tiers, packages, agreements
  • Xero invoicing
  • Confirmation + reminder emails
  • Location enforcement (server-side, returns 400 if venue not in allowedLocations)

Concept: Podcast studio bookings as a development journey — same pipeline as mentoring (Kakano → Tipu → Ora). Access creates impact. Track development, not just output.

🎤 Venue Config

Not pure feelings. Not pure numbers. Both.
Tangible (did you publish? what worked?) + Meaningful (is this becoming real for you? are doors opening?).

🧠 Capability Building Framework

Core concept: Everything Reserve Tāmaki delivers is capability building — at different scales, through different formats.

Format Description RTMKD Examples Platform Module
1:1 — Mentoring Deep, sustained, personalised support Ra/Kim + mentee, regular sessions Mentoring module
1:Few — Programmes Structured, cohort-based, group workshops Creators workshops, How to Podcast, Level Up Programmes + Creator Journey
1:Many — Broadcast Scalable, accessible, wider reach Newsletters, YouTube series (future) Comms module (in development)
Reporting implication: Mentoring (1:1) and Programmes (1:Few) are the same kaupapa — different format. Report together as "Capability Delivered" with breakdown by format.

Placemaking vs Capability: Space Use = placemaking (enabling others). Mentoring + Programmes = direct capability delivery. Both matter, both tell different stories.
📊 Reporting Definitions

Use these consistently across all quarterly and monthly reports.

Activations

Everything that happened in the space — external use + internal delivery. The headline number.
Includes: Hub Activities, Programmes, Drop-ins, External Events, Venue Hires, Space Use
Excludes: Meetings, Catch-ups, Planning, Mentoring (tracked separately)

Space Use (external)

Subset of Activations. External orgs and individuals using the space.
Answers: who came to use what we've built?

Internal Delivery

Subset of Activations. Programmes and workshops delivered by Reserve Tāmaki staff.
Answers: what did we actively deliver?

Mentoring & Programmes

Tracked separately from Activations. Structured capability delivery — 1:1 mentoring + 1:Few group programmes. Same kaupapa, different format. Report together.

Foot Traffic

Total people through the space, tracked by Kim via cameras. Source of truth = daily_foot_traffic table.

Quarter Note ⚠

Reserve Tāmaki = calendar year quarters (Q1 = Jan–Mar). Auckland Council = financial year (our Q1 = their Q3 FY25/26). Always clarify per audience.

📋 Reporting Standards TO SPEC — NOT BUILT
Quarterly Report (external — funders, council)
  • Format: Structured data table + narrative
  • Numbers: Activations, Capability Building, Space Use, Foot Traffic, Community Snapshot
  • Sections: Delivery Numbers → Community Snapshot → Space Use → Updates → In Their Words → Planned Next Quarter
  • Audience: Auckland Council Māori Outcomes (primary)
  • Cadence: Jan–Mar (Q1), Apr–Jun (Q2), Jul–Sep (Q3), Oct–Dec (Q4)
  • Council labels differ — our Q1 = their Q3 FY25/26. Always clarify.
  • Output: PDF via HTML print or Google Doc
Monthly Report (internal + light external)
  • Format: More narrative, photo-led, less formal tables
  • Numbers: Monthly counts only (no quarterly rollup)
  • Sections: Highlights, key people, space use, what's coming
  • Photos: Include 2–3 images from the month
  • Audience: Lighter-touch funders, GridAKL, internal team
  • Cadence: Monthly, before last working day of month
  • Output: Google Doc or email body
Key distinction: Quarterly = accountability and outcomes evidence. Monthly = story of the month, builds funder relationships. Both pull from the same RTMKD data.
