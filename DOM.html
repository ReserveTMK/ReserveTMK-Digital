🎙️ Creator Journey

NEW MODULE

Concept: Podcast studio bookings as a development journey — same pipeline as mentoring (Kakano → Tipu → Ora). Access creates impact. Track development, not just output.

Venue Config

  • Black Set — max 4 people: 1 host + max 2 guests + 1 live editor
  • Lounge Set — max 3 people: 1 host + max 1 guest + 1 live editor
  • Location: Podcast Studio
  • Strict studio use — no coworking, no extra guests

Onboarding (first studio booking)

Branching question: Is this a new podcast or one you already have live?

New podcast: Name | Kaupapa | Goal | Target audience | What's stopping you?

Existing podcast: Name | Platform link | Episode count | Following | Goal from here

Per-session booking form

  • Episode title (required)
  • What's this episode about? (required)
  • Host name — pre-filled, editable (required)
  • Guest names — required, max varies by venue
  • About each guest (optional)
  • Live editor attending — yes/no
  • Te reo content — yes/no
  • Guest from Tāmaki community — yes/no

Rebooking check-in (on next booking, 48hrs+ after last session)

  • How did it go? — 5-star scale (required)
  • Did you publish the episode? — Yes / Not yet / No (required)
  • What worked well? — multi-select: Content flow | Guest chemistry | Audio quality | Studio setup | Something else
  • Any challenges? — multi-select: Technical issues | Guest didn't show | Content direction | Time pressure | Nothing major
  • One thing you'd do differently? — short optional text
  • Is this becoming something real for you? — simple scale (required)
  • Are doors opening because of this podcast? — yes / starting to / not yet (required)

Journey stages (same as innovator pipeline)

  • Kakano — first sessions, irregular, figuring it out
  • Tipu — consistent, episodes releasing, finding voice
  • Ora — established, community impact, collaborations forming

Design Principle — Check-in Approach

Not pure feelings. Not pure numbers. Both.

Tangible: Did you publish? What worked? What didn't? Did they show up? Studio/gear issues?

Meaningful: Is this becoming real for you? Are doors opening? Is confidence growing? Are you connecting people through this?

The test: would a real person feel like this question respects both their craft AND their humanity? If yes, keep it.

This principle applies across all progress tracking — Creator Journey, Mentoring growth surveys, programme check-ins.

🧠 Capability Building Framework

Core concept: Everything Reserve Tāmaki delivers is capability building — at different scales, through different formats.

Format Description Current RTMKD Examples Platform Module
1:1 — Mentoring Deep, sustained, personalised support Ra/Kim + mentee, regular sessions Mentoring module
1:Few — Programmes Structured, cohort-based, group workshops Creators workshops, How to Podcast, Level Up sessions Programmes module + Creator Journey
1:Many — Broadcast Scalable, accessible, wider reach — webinars, content series Newsletters, YouTube series (future), podcast content Comms module (in development)

Reporting implication: Mentoring sessions (1:1) and Programmes (1:Few) are the same kaupapa — different delivery format. Report them together as "Capability Delivered" with breakdown by format.

Placemaking vs Capability: Space Use (activations by external orgs) is placemaking — enabling others. Mentoring + Programmes is direct capability delivery. Both matter, both tell different stories.

📊 Reporting Definitions

Use these consistently across all quarterly and monthly reports.

Activations

Everything that happened in the space — both external use and internal delivery combined. The headline number.
Includes: Hub Activities, Programmes, Drop-ins, External Events (delivered by us), Venue Hires, Space Use by external orgs
Excludes: Meetings, Catch-ups, Planning sessions, Mentoring sessions (tracked separately)

Space Use (external)

Subset of Activations. External organisations and individuals who used the space — hirers, community groups, creators.
Answers: who came to use what we've built?

Internal Delivery

Subset of Activations. Programmes and workshops delivered by Reserve Tāmaki staff.
Answers: what did we actively deliver?

Mentoring & Programmes

Tracked separately from Activations. Structured capability delivery — 1:1 mentoring sessions + 1:Few group programmes.
Same kaupapa, different format. Report together.

Foot Traffic

Total people through the space, tracked by Kim via cameras. Source of truth = platform daily_foot_traffic table.

Quarter note

Reserve Tāmaki uses calendar year quarters (Q1 = Jan-Mar). Auckland Council uses financial year quarters (our Q1 = their Q3 FY25/26). Always clarify which quarter label is being used per audience.

📋 Reporting Standards

TO SPEC — NOT BUILT YET

Quarterly Report (external — funders, council)

  • Format: Structured data table + narrative sections
  • Numbers: Activations, Capability Building, Space Use, Foot Traffic, Community Snapshot
  • Sections: Delivery Numbers → Community Snapshot → Space Use → Updates → In Their Words → Planned Next Quarter
  • Audience: Auckland Council Māori Outcomes (primary), other funders
  • Cadence: Quarterly — Jan-Mar (Q1), Apr-Jun (Q2), Jul-Sep (Q3), Oct-Dec (Q4)
  • Note: Council labels differ — our Q1 = their Q3 FY25/26. Always clarify.
  • Output: PDF via HTML print or Google Doc

Monthly Report (internal + light external)

  • Format: More narrative, photo-led, less formal tables
  • Numbers: Monthly counts only (no quarterly rollup)
  • Sections: Highlights narrative, key people, space use, what's coming
  • Photos: Include 2-3 images from the month
  • Audience: Lighter-touch funders, GridAKL (monthly obligation), internal team
  • Cadence: Monthly, due before last working day of month
  • Output: Google Doc or email body

Key distinction

Quarterly = accountability and outcomes evidence. Monthly = story of the month, builds relationship with funders. Both pull from the same RTMKD data but serve different purposes.
