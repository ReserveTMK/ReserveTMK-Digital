-----------------------------------------
PART 3: “DEBRIEF RECONCILIATION” (XERO-LIKE WORKFLOW)
-----------------------------------------

Goal:
Reframe debrief completion as “reconciling” delivery, similar to how Xero reconciles transactions.
Create a prominent “To Be Debriefed” queue on the dashboard.

Core Concept:
- Events / Programme sessions that require debriefing should enter a queue until reconciled.
- “Reconcile” is the primary action, not “Add Debrief”.

Data Requirements:
Ensure these fields exist and are used:
- calendar_event.requires_debrief (boolean)
- debrief.status (draft → pending_review → confirmed)
- link between event/programme and debrief (event_id or programme_session_id)

Queue Logic (“To Be Debriefed”):
Show items where:
- requires_debrief = true
AND
- event_date < now()
AND
- no confirmed debrief linked

Each row shows:
- date
- type (programme / mentoring / meeting)
- title
- linked contacts count (if any)
- status badge (overdue / due / pending)

Primary CTA:
- “Reconcile” button

Reconcile Flow (single screen):
When clicked:
1) Record or attach debrief (voice or text)
2) Confirm linked contacts/groups
3) Review taxonomy matches (accept/edit/reject)
4) Confirm and mark debrief status = confirmed
5) Mark item as reconciled (removed from queue)

Optional:
- “No Debrief Required” action with required reason (stored)

Dashboard Changes:
Add a prominent card near the top:
- “To Be Debriefed” with a count badge
- Clicking opens full queue view

Acceptance Criteria:
- Dashboard clearly shows the count of items needing debrief.
- Reconcile flow is fast (max ~1 minute per item).
- Once confirmed, the item disappears from the queue.
- No manual hunting through calendar required.
