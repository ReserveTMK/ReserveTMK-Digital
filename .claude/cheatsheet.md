# Claude Cheat Sheet — What to Say

## When something's broken
"The booking page shows $0 for community bookings"
"This screenshot has a white screen — debug it"
"Bookings aren't sending confirmation emails"
→ Describe the symptom. I find the cause.

## When you need a report
"Write the March Council report" — full end-to-end
"How are we tracking against Council deliverables?" → `/funder-pulse council`
"Is the data clean enough for the report?" → `/month-close`
"Give me the numbers for March" → `/report 2026-03`

## When you need to prep
"I have a meeting with Nick Turoa — what should I know?"
"Prep me for the Foundation North conversation"
"What's our story for the last quarter?"
→ Tell me who and why. I pull data + draft talking points.

## When you want to build
"Bookers need to see their payment status"
"Add a way for Ra to book on behalf of a mentee"
→ Describe what the user should be able to do. I plan and build.

## When you want to check in
"What's this week look like?" → `/week-ahead`
"What needs my attention?" → `/deadline-watch`
"Are we on track?" → `/funder-pulse`
"What should we build next?" → `/suggest [area]`

## When you're not sure
"/help-me" → I'll ask what you're trying to do and point you to the right thing.

## Quick commands
`/month-close` — is data clean for the report?
`/week-ahead` — what's coming this week?
`/funder-pulse` — are we delivering on agreements?
`/suggest spaces` — what should we improve?
`/absorb` — process files from ~/Drop
`/db [query]` — run a database query

## How to get the best out of me
- **Say why, not just what** — "for the Council report" changes my output
- **Describe the problem, not the fix** — let me find the root cause
- **Say "note:" when something matters** — I save it to memory
- **Say "handle the whole thing"** — I can chain skills end-to-end
- **Start with intent** — "I need to impress Nick Turoa" > "pull Māori data"
